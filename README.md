Простой CRUD интерфейс: список пользователей и товаров

В качестве API Endpoint был использован mockAPI (https://mockapi.io/clone/69d71c379c5ebb0918c7025d)
Endpoint указывается в .env VITE_API_BASE
Использованы компоненты Nuxt UI для элементов UI

Install: pnpm install
Start dev: pnpm run dev
Build: pnpm run build
Lint: pnpm run lint

Было решено разделить интерфейс на 4 компонента: CrudToolbar, ProductsTable, UsersTable и RecordFormModal
Все функции и логика работы разделены на composables: useCrudApi.ts, useCrudFilters.ts, useCrudFormState.ts и useCrudManager.ts, который отдает интерфейс в App.vue
Типы данных для полей таблиц, и для полей формы указаны в types/crud.ts

Реализованы состояния загрузки в таблицах, и кнопке Save в модальном окне
Состояния ошибок: при ошибках показывается сообщение на странице, при ошибках действий появляются toast об ошибках.
Для пустой таблицы и пустого результата поиска показывается отдельный empty state.
Реализованы placeholder-ы вместо невалидных данных.
В представленной таблице реализованы все CRUD функции: добавление, удаление, просмотр и изменение строк.
Есть возможность поиска (фильтрации) по таблице и сортировки строк по нажатию на столбец.
Модаль для добавления/изменения записей имеет базовую валидацию данных.

Что можно было улучшить:

- продвинутый поиск (fuzzy search, теги, regex)
- pagination и серверный поиск
- i18n интерфейса и единиц измерения
- возможность выбора и массового удаления/изменения записей (требует соответствующих методов в API)
- навигация с клавиатуры, aria labels, focus management
- Schema-driven validation для end-to-end type safety
